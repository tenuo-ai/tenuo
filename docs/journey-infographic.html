Tenuo User Journey

From setup to enforcement: how to integrate Tenuo into your AI agent infrastructure

Setup

1. Deploy Control Plane

Start the Tenuo Control Plane. It generates its root keypair and becomes your trust anchor.

# Start Control Plane with environment variables $ export TENUO_BIND_ADDR=0.0.0.0:8080 $ export TENUO_ENROLLMENT_TOKEN=$(openssl rand -hex 32) $ export TENUO_SECRET_KEY=$(openssl rand -hex 32) # Optional: use existing key $ tenuo-control # Output: Control Plane public key Public Key: f32e74b5b856...

Enrollment

3. Enroll Agents

Orchestrators generate keypairs locally and enroll with the Control Plane to receive root warrants.

# Orchestrator generates its own keypair let keypair = Keypair::generate(); # Enrolls with Control Plane POST /v1/enroll { "public_key": "a1b2c3...", "signature": "...", // PoP "constraints": { "cluster": "staging-*" } }

Delegation

4. Attenuate & Delegate

Parent agents attenuate warrants before delegating to children. Constraints can only shrink.

# Narrow to specific cluster let child = parent.attenuate() .constraint("cluster", Exact::new("staging-web")) .constraint("cost", Range::new(0.0, 100.0)) .build(&worker_keypair)?;
Verification

5. Extract & Authorize

Gateway extracts values from the request, verifies the warrant chain, and checks constraints.

# Request arrives POST /api/v1/clusters/staging-web/scale X-Tenuo-Chain: <base64 warrant chain> X-Tenuo-PoP: <signature> { "spec": { "replicas": 5 }, "metadata": { "estimatedCost": 50.0 }} # Extracted constraints cluster: "staging-web" ‚úì matches cost: 50.0 ‚úì within range
